# UI: правила отображения (темы, масштаб, панели)

Этот документ — **продуктовые требования/критерии приёмки** к отображению интерфейса.

Подробные инженерные правила (токены, layout-правила и т.п.) описаны в:

- [README.ui-rules.md](../../README.ui-rules.md)

## 1) Темы (light/dark/system)

Требования:

- Приложение поддерживает режимы: `system`, `light`, `dark`.
- По умолчанию используется `system`.
- Выбор пользователя сохраняется и применяется при следующем открытии приложения.

Критерии приёмки:

- Переключение темы не ломает layout (нет скачков/переполнений).
- Сохраняется читабельность текста и контрастность элементов управления.

## 2) Масштаб (zoom) и доступность

Требования:

- UI остаётся работоспособным при зуме браузера в диапазоне примерно 80%–200%.
- Важные действия остаются доступными (кнопки не «уезжают» за экран/не становятся недоступными).

Критерии приёмки:

- На 200% нет критически важных элементов, которые невозможно нажать/достать без хака.
- Фокус-обводка видима при навигации с клавиатуры.

## 3) Панель «Свойства» и внутренние окна

Требования:

- На десктопе панель «Свойства» закреплена справа и имеет собственный вертикальный скролл.
- На узких экранах панель «Свойства» не должна постоянно занимать ширину — превращается в overlay/drawer по запросу.
- Внутренние окна (вложенные панели/выборы/редакторы) не выходят за границы viewport.

Критерии приёмки:

- Центральный контент остаётся основным и не становится неюзабельным из‑за панели.
- При изменении размера окна браузера layout перестраивается без «ломания» верстки.

## 4) Минимальный чек перед релизом

Проверить визуально на одной и той же странице:

- темы: `light`/`dark`/`system`
- ширины окна: ~320px, ~768px, ~1024px, ~1440px
- zoom: 80% и 200%
